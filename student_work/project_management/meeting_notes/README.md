## Meeting Types

### 1. Initial Consultation
**Purpose:** Understand the problem and stakeholder needs

**Topics:**
- Problem introduction
- Stakeholder background
- Initial requirements
- Success criteria
- Project scope

### 2. Requirements Gathering
**Purpose:** Detailed requirements collection

**Topics:**
- Functional requirements
- Non-functional requirements
- User stories
- Priority discussions
- Constraints and limitations

### 3. Design Review
**Purpose:** Get feedback on design plans

**Topics:**
- Architecture review
- UI/UX feedback
- Technical approach validation
- Design approval

### 4. Progress Demonstrations
**Purpose:** Show development progress

**Topics:**
- Features completed
- Demo of working code
- Feedback collection
- Next iteration planning

### 5. Testing Sessions
**Purpose:** User acceptance testing

**Topics:**
- Feature testing
- Bug identification
- Usability feedback
- Acceptance validation

### 6. Final Evaluation
**Purpose:** Project completion review

**Topics:**
- Success criteria evaluation
- Overall satisfaction
- Lessons learned
- Future recommendations

## Meeting Preparation Checklist

### Before Meeting
- [ ] Schedule meeting time
- [ ] Prepare agenda
- [ ] Gather materials to show/discuss
- [ ] Prepare questions
- [ ] Test demo if showing code
- [ ] Bring notebook for notes

### During Meeting
- [ ] Take detailed notes
- [ ] Record decisions
- [ ] Clarify unclear points
- [ ] Get feedback documented
- [ ] Confirm action items
- [ ] Schedule next meeting

### After Meeting
- [ ] Write up notes immediately
- [ ] Distribute notes if needed
- [ ] Complete action items
- [ ] Follow up on questions
- [ ] Update project documentation

## Best Practices

### 1. Be Prepared
- Know what you want to discuss
- Have materials ready
- Prepare questions in advance
- Review previous meeting notes

### 2. Active Listening
- Pay attention
- Take good notes
- Ask clarifying questions
- Confirm understanding

### 3. Document Thoroughly
- Record all decisions
- Note all feedback
- Track action items
- Date everything

### 4. Follow Up
- Complete action items
- Send thank you if appropriate
- Share updates
- Schedule next meeting

### 5. Professional Conduct
- Be on time
- Be respectful
- Be prepared
- Be responsive

```markdown
- [ ] Task 1
- [ ] Task 2

```

## Tips

- **Write notes during or immediately after**: Don't trust memory
- **Be specific**: Include details
- **Note decisions**: These are important
- **Track action items**: Who does what by when
- **Be professional**: Notes may be reviewed
- **Share when appropriate**: Keep stakeholders informed

## Common Mistakes

- ❌ Not taking notes
- ❌ Writing notes days later
- ❌ Vague or incomplete notes
- ❌ Missing action items
- ❌ Not following up
