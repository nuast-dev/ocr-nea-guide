This folder contains **screenshots** documenting your OCR NEA project.

## What to Include

### Application Screenshots
- All major features
- Different screens/pages
- User workflows
- Before and after improvements
- Progressive development stages

### Development Process
- Code editor showing your work
- Development environment
- Version control (Git) activity
- Development tools in use

### Testing Evidence
- Tests being executed
- Test results
- Pass/fail indicators
- Bug demonstrations
- Before/after bug fixes

### Stakeholder Interactions
- Demonstrations to stakeholder
- Feedback sessions
- User acceptance testing
- Stakeholder using the system

## Screenshot Guidelines

### Quality
- Clear and readable text
- Appropriate resolution (not blurry)
- Full screen or relevant portion
- Good contrast and visibility

### Content
- Show relevant information
- Include context (window titles, dates)
- Highlight important elements if needed
- Avoid cluttered backgrounds

### Naming Convention

Good examples:
- `2026-02-15_homepage_v1.png`
- `login_screen_mobile_view.png`
- `test_results_unit_tests.png`
- `stakeholder_demo_search_feature.png`
- `bug_fix_before_validation_error.png`
- `bug_fix_after_validation_working.png`

Bad examples:
- `Screenshot1.png`
- `IMG_20260215.png`
- `Screen Shot.png`
- `final_FINAL.png`

## How to Take Good Screenshots

### Windows
- **Full screen**: PrtScn key
- **Active window**: Alt + PrtScn
- **Snipping Tool**: For selective areas

### Mac
- **Full screen**: Cmd + Shift + 3
- **Selection**: Cmd + Shift + 4
- **Window**: Cmd + Shift + 4, then Space

### Linux
- **Full screen**: PrtScn
- **Selection**: Shift + PrtScn
- Or use screenshot tools like GNOME Screenshot

## Tips

- **Capture as you go**: Don't wait until the end
- **Be consistent**: Use same format throughout
- **Include dates**: Use YYYY-MM-DD in filenames
- **Annotate if needed**: Add arrows or highlights
- **Keep originals**: Don't over-edit
- **Be selective**: Quality over quantity

## Privacy Considerations

- Remove sensitive information (passwords, API keys)
- Anonymize personal data if necessary
- Blur out confidential information
- Consider GDPR requirements

## File Formats

- **PNG**: Best for screenshots (lossless)
- **JPG**: Use if file size is a concern
- Avoid proprietary formats
