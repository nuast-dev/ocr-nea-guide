## Organization Tips

### 1. Keep Everything Organized
- Use clear folder names
- Use consistent file naming conventions
- Keep related files together
- Date files when appropriate (YYYY-MM-DD format)

### 2. Version Control
- Use Git to track changes
- Make regular commits with meaningful messages
- Don't commit sensitive data (passwords, API keys)
- Use `.gitignore` for build artifacts and dependencies

### 3. Backup Regularly
- Keep backups in multiple locations
- Use cloud storage (Google Drive, OneDrive, etc.)
- Test your backups periodically
- Don't rely on a single copy

### 4. Evidence Collection
- Take screenshots as you work
- Save all stakeholder communications
- Keep notes from meetings
- Document problems and solutions as they happen

## File Naming Conventions

Good examples:
- `2025-09-15_stakeholder_interview_notes.pdf`
- `homepage_wireframe_v2.png`
- `login_function_unit_tests.py`
- `search_algorithm_pseudocode.md`

Bad examples:
- `Document1.pdf`
- `Screenshot.png`
- `final_FINAL_v3_actualfinal.docx`
- `stuff.txt`

## What Goes Where?

### Documentation
All your written work:
- Analysis reports
- Design documents
- Development log entries
- Testing documentation
- Evaluation reports

### Code
All your programming work:
- Source code files
- Test files
- Configuration files
- Build scripts
- README files

### Evidence
Supporting materials:
- Screenshots of your application
- Screenshots of testing
- Stakeholder interview transcripts
- Email correspondence
- Meeting notes with stakeholders

### Research
Background research:
- Analysis of existing solutions
- Technical research notes
- Library/framework comparisons
- Academic papers or articles referenced

### Project Management
Organizational materials:
- Timeline and milestones
- Task lists and progress tracking
- Meeting agendas and minutes
- Risk assessments
